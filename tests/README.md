# OPFS Web UI — Playwright E2E

End-to-end tests for the `/opfs/` UI shipped by NVGD.

Test design and per-scenario rationale live in [`PLAN.md`](./PLAN.md).
This file is the operational manual: how to run the tests, how the
folders are organised, and what assumptions about the environment they
make.

## Prerequisites

1. **Go ≥ 1.24** with CGO enabled. Used to build `nvgd`.
2. **Node ≥ 20** (Playwright requires a recent LTS).
3. **Playwright browsers installed** — once per machine:

   ```bash
   npx playwright install --with-deps chromium firefox
   ```

## Running locally

The tests assume an `nvgd` instance is already listening on
`http://127.0.0.1:9280`. We deliberately do *not* let Playwright manage
the server because Go builds belong to the project's own Makefile.

In one terminal:

```bash
make build                       # produces ./nvgd
./nvgd -c tests/fixtures/nvgd.conf.yml
```

In another:

```bash
npx playwright test              # both projects: chromium + firefox
npx playwright test --project=chromium
npx playwright test tests/specs/01-initial-load.spec.ts
npx playwright test --ui          # interactive runner
```

The base URL is overridable:

```bash
NVGD_BASE_URL=http://127.0.0.1:18080 npx playwright test
```

After a run, the HTML report opens with:

```bash
npx playwright show-report
```

## CI

`.github/workflows/playwright.yml` builds nvgd, starts it in the
background, waits for the port, then runs `npx playwright test`. The
HTML report is uploaded as an artifact (`playwright-report`).

## Conventions

- Each test starts by navigating to `/opfs/` and calling
  `clearOPFS(page)` to ensure an empty origin storage.
- Mutations performed by the UI are observed via Page Object locators
  (`expect(opfs.row('foo')).toBeVisible()`); raw OPFS state can be read
  back with the helpers in `helpers/opfs.ts`.
- Confirm/alert dialogs are always wired up with `captureDialog`,
  `recordDialogs`, or `autoAcceptDialogs` from `helpers/dialogs.ts`. A
  test that triggers a dialog without registering a handler will hang.
- WebKit is excluded from the project list. The File System Access API
  parts that are tested by `08-save-as.spec.ts` are Chromium-only and
  are explicitly skipped on Firefox.

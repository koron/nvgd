# OPFS UI テスト計画

**対象:** `/opfs/` プロトコルハンドラーの Web UI  
**作成日:** 2026-05-27  
**対象ファイル:** `protocol/opfs/assets/index.html`, `protocol/opfs/assets/main.js`

---

## 概要

OPFS (Origin Private File System) UIは、ブラウザのプライベートファイルシステムを操作するインターフェースです。以下の機能を提供します。

- ディレクトリ一覧の表示・ナビゲーション
- ディレクトリの作成
- ローカルファイルのアップロード
- シンプルエディタによるファイル作成・編集
- URLからのファイルダウンロード
- ファイル・ディレクトリの削除
- DuckDB連携

---

## テストシナリオ一覧

### TC-01: 初期表示

**概要:** ページを初めて開いたときの表示を確認する

**前提条件:** OPFS に何もファイルが存在しない（初期状態）

**手順:**

1. `/opfs/` にアクセスする
2. ページタイトルが `OPFS: (Root)/` であることを確認する
3. ヘッダーにパンくずリスト `(Root)` が表示されていることを確認する
4. ファイル一覧テーブルのヘッダー（Name / Type / Size / Modified At / Actions）が表示されていることを確認する
5. ファイル行が0件であることを確認する
6. `Reload` ボタンが有効であることを確認する
7. `Delete` ボタンが無効（disabled）であることを確認する
8. `DuckDB` ボタンが無効（disabled）であることを確認する

**期待結果:** 空のディレクトリ一覧が正常に表示される

---

### TC-02: ディレクトリの作成

**概要:** 「Make directory」セクションから新しいディレクトリを作成する

**前提条件:** ルートディレクトリを表示している

**手順:**

1. 「Make directory」の入力欄に `testdir` と入力する
2. `Create new directory` ボタンをクリックする
3. ファイル一覧に `testdir/` が追加されることを確認する
4. Type が `dir`、Size が `(N/A)`、Modified At が `(N/A)` であることを確認する
5. 入力欄がクリアされることを確認する

**期待結果:** ディレクトリが作成されて一覧に反映される

---

### TC-03: ディレクトリ作成 — 名前が空のとき

**概要:** ディレクトリ名を入力せずに作成しようとするとアラートが出ることを確認する

**前提条件:** ルートディレクトリを表示している

**手順:**

1. 「Make directory」の入力欄を空のままにする
2. `Create new directory` ボタンをクリックする
3. `"Need directory name"` のアラートが表示されることを確認する
4. アラートを閉じる
5. ファイル一覧に変化がないことを確認する

**期待結果:** バリデーションエラーアラートが表示され、ディレクトリは作成されない

---

### TC-04: ディレクトリへのナビゲーション

**概要:** ディレクトリ名をクリックしてサブディレクトリへ移動する

**前提条件:** TC-02 を実行済みで `testdir/` が存在する

**手順:**

1. 一覧の `testdir/` リンクをクリックする
2. ページタイトルが `OPFS: .../testdir/` に変わることを確認する
3. ヘッダーのパンくずリストに `(Root)` と `testdir` が表示されていることを確認する
4. `(Root)` のパンくずリンクをクリックする
5. ルートに戻ることを確認する

**期待結果:** ディレクトリ移動とパンくずナビゲーションが正しく動作する

---

### TC-05: ブラウザの「戻る」ボタンによるナビゲーション

**概要:** ブラウザのヒストリー（popstate）が正しく機能することを確認する

**前提条件:** TC-04 を参考に `testdir/` へ移動した状態

**手順:**

1. `testdir/` へ移動する
2. さらに子ディレクトリ `childdir` を作成して移動する
3. ブラウザの「戻る」ボタンを押す
4. `testdir/` の一覧に戻ることを確認する
5. 再度「戻る」を押す
6. ルートに戻ることを確認する

**期待結果:** ブラウザのヒストリーと連動してナビゲーションが正しく動作する

---

### TC-06: ローカルファイルのアップロード

**概要:** 「Upload a local file」セクションからファイルをアップロードする

**前提条件:** ルートディレクトリを表示している

**手順:**

1. 「Upload a local file」の `input[type=file]` でテスト用ファイル（例: `hello.txt`）を選択する
2. 「Upload as」の入力欄に `hello.txt` が自動入力されることを確認する
3. `Upload` ボタンが有効になることを確認する
4. `Upload` ボタンをクリックする
5. アップロード成功アラートが表示されることを確認する
6. アラートを閉じる
7. ファイル一覧に `hello.txt` が表示されることを確認する
8. Type が `file`、Size が正しいバイト数であることを確認する
9. 「Upload as」の入力欄がクリアされることを確認する

**期待結果:** ファイルが正常にアップロードされて一覧に表示される

---

### TC-07: アップロード — 同名ファイルの上書き確認

**概要:** 同名ファイルをアップロードしようとすると確認ダイアログが表示される

**前提条件:** TC-06 を実行済みで `hello.txt` が存在する

**手順:**

1. 同名の `hello.txt` ファイルを再度選択してアップロードしようとする
2. `Upload` ボタンをクリックする
3. 上書き確認ダイアログ `'Are you sure to overwrite a file with the same name, "hello.txt"?'` が表示されることを確認する
4. 「キャンセル」をクリックする
5. ファイルが上書きされていないことを確認する
6. 再度アップロードして今度は「OK」をクリックする
7. ファイルが更新されることを確認する

**期待結果:** 上書き保護の確認ダイアログが正しく動作する

---

### TC-14: ファイルの選択とDeleteボタンの有効化

**概要:** ファイルを選択すると Delete・DuckDB ボタンが有効になる

**前提条件:** ファイルが1件以上存在する

**手順:**

1. `Delete` ボタンが無効（disabled）であることを確認する

3. 任意のファイルのチェックボックスをオンにする

### TC-15: 全選択チェックボックス

**概要:** ヘッダーの「全選択」チェックボックスが正しく動作する

**前提条件:** ファイルが2件以上存在する

5. すべての行のチェックボックスがオフになることを確認する

7. 全選択チェックボックスが不定状態（indeterminate）になることを確認する

**期待結果:** 全選択・全解除・不定状態が正しく動作する

6. 再度チェックして `Delete` → 今度は「OK」をクリックする
7. ファイルが一覧から消えることを確認する

**期待結果:** 確認ダイアログを経てファイルが正しく削除される

### TC-18: URLからのファイルダウンロード

**概要:** 「Download (NVGD) URL」セクションでURLからファイルを保存できる

**前提条件:** nvgd サーバーが起動しており、アクセス可能な URL がある

**手順:**

1. 「Download (NVGD) URL」の URL 入力欄に `http://` で始まる有効な URL を入力する
2. `Download` ボタンが無効のままであることを確認する（名前未入力のため）
3. 「Download as」欄にファイル名（例: `downloaded.txt`）を入力する
4. `Download` ボタンが有効になることを確認する
5. `Download` ボタンをクリックする
6. ファイルが OPFS に保存されて一覧に表示されることを確認する
7. `Clear` ボタンをクリックして入力欄がクリアされることを確認する

**期待結果:** URLからファイルをダウンロードして OPFS に保存できる

### TC-19: ダウンロード — 無効な URL

**概要:** `http:` または `https:` で始まらない URL は Download ボタンが無効のまま

**前提条件:** ルートディレクトリを表示している

**手順:**

1. URL 欄に `ftp://example.com/file.txt` と入力する
2. 「Download as」欄に `test.txt` を入力する
3. `Download` ボタンが無効（disabled）のままであることを確認する
4. URL 欄に `https://example.com/file.txt` に書き換える
5. `Download` ボタンが有効になることを確認する

**期待結果:** http/https 以外のプロトコルではダウンロードボタンが無効になる

### TC-20: DuckDB 連携

**概要:** 対応形式のファイルを選択して DuckDB で開ける

**前提条件:** `.csv`、`.parquet`、`.xlsx`、`.json` などの対応形式ファイルが存在する

**手順:**

1. 対応ファイル（例: `data.csv`）のチェックボックスをオンにする
2. `DuckDB` ボタンが有効になることを確認する
3. `DuckDB` ボタンをクリックする
4. 新しいタブで `/duckdb/` が開くことを確認する
5. URL に `opfs=` パラメーターが含まれることを確認する

**期待結果:** 対応ファイルを DuckDB シェルで開くことができる

---

### TC-21: DuckDB 連携 — 非対応形式のファイル

**概要:** DuckDB 非対応形式のファイルのみ選択した場合、DuckDB ボタンの挙動を確認する

**前提条件:** `.txt` などの非対応形式ファイルのみが存在する

**手順:**

1. 非対応ファイル（例: `note.txt`）のチェックボックスをオンにする
2. `DuckDB` ボタンが有効になることを確認する（選択があるため有効化される）
3. `DuckDB` ボタンをクリックする
4. DuckDB がクエリなし（またはSHOW TABLESのみ）の状態で開くことを確認する

**期待結果:** 非対応形式ファイルはDuckDB のビューが作成されないが、シェルは開く

---

### TC-22: ディレクトリの再帰削除

**概要:** ファイルを含むディレクトリをまとめて削除できる

**前提条件:** `testdir/` の中にファイルが1件以上存在する

**手順:**

1. `testdir/` のチェックボックスをオンにする
2. `Delete` ボタンをクリックする
3. 確認ダイアログに `testdir/`（スラッシュつき）が表示されることを確認する
4. 「OK」をクリックする
5. `testdir/` が一覧から消えることを確認する（中身ごと削除）

**期待結果:** ディレクトリが再帰的に削除される

## テスト対象外（スコープ外）

- `Save as` アクション（`window.showSaveFilePicker` を使用しており、ブラウザのネイティブダイアログのためPlaywrightでの制御が困難）
- 64KiB 以上のファイルに対する Edit アクションの非表示（実環境での大容量ファイル準備が必要）
- ファイルロック時のエラーメッセージ（`NoModificationAllowedError`）

---

## テスト実施環境

- **対象ブラウザ:** Chromium（OPFSはFirefoxでもサポートされるが、DuckDB連携等の確認はChromiumを推奨）
- **サーバー:** nvgd をローカル起動（デフォルト `http://127.0.0.1:9280`）
- **テストフレームワーク:** Playwright (TypeScript)
